## Version 1.1.0 (2026-03-08)

### 功能更新
1. **实现动态赛事报告加载**
   - 赛程页面中的赛果链接现在会生成带参数的 URL
   - 赛事报告页面会根据 URL 参数自动加载对应的 JSON 文件

2. **文件命名规范**
   - 为每一场比赛创建独立的 JSON 文件
   - 文件名格式：`日期-比赛类型-轮次.json`
   - 示例：`2026-03-07-中超-第1轮.json`

3. **数据格式保持一致**
   - 所有赛事报告 JSON 文件保持与 `match_report.json` 相同的格式
   - 包含完整的比赛信息、球员数据、统计数据等

4. **用户体验优化**
   - 保持原来的页面格式和显示效果
   - 支持直接从赛程页面跳转到对应赛事报告
   - 提供清晰的错误提示当找不到对应的赛事报告时

### 文件更改
- `index.html`：修改赛程组件，生成带参数的链接
- `match-report.html`：实现动态数据加载逻辑
- `data/2026-03-07-中超-第1轮.json`：创建第一场比赛的赛事报告文件

### 使用方式
1. 为每一场比赛创建对应的 JSON 文件，按照指定格式命名
2. 点击赛程中的赛果链接，系统会自动加载对应的赛事报告
3. 所有赛事报告都将保持与原来相同的格式和风格
