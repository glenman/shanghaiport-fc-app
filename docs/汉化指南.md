# 上海海港足球俱乐部数据汉化指南

## 概述

本文档详细说明上海海港足球俱乐部比赛数据的汉化处理流程，包括脚本使用方法、翻译规则和最佳实践。

## 已完成的汉化工作

### 1. 数据文件处理

**处理范围**：`public/data/history/2024/` 目录下的30个JSON文件

**统计成果**：

- 处理文件数：30个
- 翻译球员名字：154处
- 更新球队名称：81处
- 扩展球队全名：29处

### 2. 主要修改内容

#### 2.1 球队信息汉化

- **队名翻译**：将英文名翻译为中文
  - `Shanghai Port` → `上海海港`
  - `Zhejiang Professional` → `浙江队`
  - `Beijing Guoan` → `北京国安`
  - 等16支中超球队
- **全名扩展**：自动扩展为完整俱乐部名称
  - `浙江队` → `浙江足球俱乐部`
  - `上海海港` → `上海海港足球俱乐部`

#### 2.2 球员名字翻译

建立了完整的中英文球员名字映射表，包括：

- 上海海港球员（如：Zhang Linpeng → 张琳芃）
- 外籍球员（如：Nemanja Čović → 内马尼亚·科维奇）
- 其他球队球员（如：Lin Liangming → 林良铭）

#### 2.3 事件描述更新

统一更新了事件描述格式：

- **进球**：`球员在X'进球，助攻球员：球员2`
- **黄牌/红牌**：`球员获得黄牌/红牌`
- **换人**：`球员_out被换下，球员替补登场`
- **乌龙球**：特殊标记OG

## 脚本说明

### 1. 综合汉化脚本 `scripts/translate-all.js`

**功能**：一次性完成所有汉化工作

```bash
node scripts/translate-all.js
```

**处理内容**：

- 翻译球员名字（首发、替补、换人、事件）
- 更新球队名称和全名
- 更新事件描述

### 2. 专项脚本

- **`scripts/update-team-names.js`**：专门更新球队名称
- **`scripts/translate.js`**：专门翻译球员名字

## 翻译映射表维护

### 添加新球员翻译

编辑 `scripts/translate-all.js` 中的 `playerTranslations` 对象：

```javascript
const playerTranslations = {
  // 添加新的翻译
  'NewPlayerName': '中文名字',
};
```

### 添加新球队翻译

编辑 `scripts/translate-all.js` 中的 `teamTranslations` 对象：

```javascript
const teamTranslations = {
  // 添加新的球队翻译
  'NewTeamName': '中文队名',
};
```

## 最佳实践

1. **数据源基准**：每个赛季的球员和球队名称更新应以 `datafile/` 目录下的CSV文件为基准：
   - `上海海港XXXX一线队大名单.csv`：用于更新上海海港球员名字
   - `上海海港XXXX一线队中超赛程.csv`：用于更新对手球队名称
2. **保持数据一致性**：所有相关文件使用相同的翻译映射表
3. **定期更新**：新赛季开始时更新翻译表
4. **测试验证**：修改后运行脚本并检查页面展示
5. **备份数据**：处理前备份原始数据文件

## 技术细节

### 数据结构

JSON文件主要包含以下结构：

- `match_info`：比赛基本信息
- `teams`：球队信息（home/away）
- `events`：比赛事件
- `statistics`：统计数据

### 事件类型

支持的事件类型：

- `goal`：进球
- `penalty_goal`：点球进球
- `own_goal`：乌龙球
- `yellow_card`：黄牌
- `red_card`：红牌
- `substitution`：换人
