# 2023赛季赛事报告优化指南

## 概述

本指南说明如何使用 `optimize_2023_reports.py` 脚本来优化2023赛季的赛事报告JSON文件。

## 功能说明

### 1. 球员姓名对照

脚本基于 `datafile/上海海港2023一线队大名单.csv` 文件创建球员姓名对照字典，支持以下两种匹配方式：

#### 按号码匹配
- 优先根据球衣号码进行精确匹配
- 适用于号码唯一的情况

#### 按拼音匹配
- 当号码匹配失败时，根据球员姓名的拼音进行模糊匹配
- 支持拼音部分匹配和包含关系
- 自动处理常见的拼音分隔符（-、·、'等）

### 3. 内容优化

脚本会自动优化以下内容：

#### 比赛信息 (match_info)
- 赛事名称汉化：`Chinese Football Association Super League` → `中国足球协会超级联赛`
- 场地名称汉化：将英文场地名称翻译为中文

#### 球队信息 (teams)
- 球队名称汉化
- 球队全名生成
- 阵容球员姓名汉化（基于号码和拼音）
- 替补球员姓名汉化
- 换人记录中的球员姓名汉化
- 国籍标准化：`Unknown` → `中国`

#### 比赛事件 (events)
- 进球、黄牌、红牌等事件中的球员姓名汉化

#### 球员统计 (player_stats)
- 球员个人统计数据中的姓名汉化

## 数据文件说明

### 球员名单文件

**文件路径：** `datafile/上海海港2023一线队大名单.csv`

**文件格式：**
```csv
号码,姓名,出生年月日,年龄,位置
1,颜骏凌,1991-01-28,35,GK
2,李昂,1993-09-15,33,DF
...
```

**注意事项：**
- 号码必须唯一
- 姓名使用中文
- 位置使用标准缩写（GK、DF、MF、FW）
- 拼音字段用于模糊匹配

### JSON文件

**文件路径：** `public/data/history/2023/*.json`

**文件命名格式：** `YYYY-MM-DD-中超-第N轮.json`

## 扩展和自定义

### 添加新球员

在脚本的 `SHANGHAI_PORT_2023_ROSTER` 字典中添加新球员信息：

```python
SHANGHAI_PORT_2023_ROSTER = {
    # 现有球员...
    46: {"name": "新球员姓名", "pinyin": "Xin Qiuyin"},
}
```

### 添加新球队

在脚本的 `TEAM_NAME_TRANSLATIONS` 字典中添加新球队对照：

```python
TEAM_NAME_TRANSLATIONS = {
    # 现有球队...
    "New Team Name": "新球队中文名",
}
```

### 自定义匹配规则

如需修改匹配逻辑，可以编辑以下函数：

- `normalize_pinyin()` - 拼音标准化
- `find_player_by_number()` - 按号码查找
- `find_player_by_pinyin()` - 按拼音查找
- `translate_team_name()` - 球队名称翻译
- `translate_player_name()` - 球员名称翻译

## 注意事项

1. **备份原文件**
   - 脚本会直接修改JSON文件
   - 建议在运行前备份 `public/data/history/2023/` 目录

2. **数据一致性**
   - 确保球员名单文件中的号码与JSON文件中的号码一致
   - 检查拼音拼写是否正确

3. **编码问题**
   - 脚本使用UTF-8编码
   - 确保所有文件使用正确的编码

4. **测试验证**
   - 运行脚本后，随机抽查几个JSON文件
   - 检查球员和球队名称是否正确汉化

## 故障排除

### 常见问题

**问题1：球员名称未汉化**
- 检查球员名单CSV文件中是否有该球员
- 检查号码是否正确
- 检查拼音是否正确

**问题2：球队名称未汉化**
- 检查 `TEAM_NAME_TRANSLATIONS` 字典中是否包含该球队
- 添加新的球队对照

**问题3：编码错误**
- 确保CSV文件使用UTF-8编码
- 确保JSON文件使用UTF-8编码
