# 更新比赛结果操作指南

本文档详细说明如何更新上海海港足球俱乐部比赛结果的操作流程，包括一线队和B队的比赛结果更新。

## 一、一线队比赛结果更新流程

### 1. 更新赛程文件

**文件路径**: `public/data/schedule.json`

**操作步骤**:

1. 打开 `schedule.json` 文件
2. 找到对应的轮次（round字段）
3. 核实给出的两个球队名称是否与文件中的名称一致
4. 更新 `result` 字段为实际比分（格式："X-Y"）
5. 将 `status` 字段从 "未开始" 或其他状态改为 "已结束"

**示例**:

```json
{
  "id": 1,
  "round": "第1轮",
  "date": "2026-03-07",
  "homeTeam": "上海海港",
  "awayTeam": "河南队",
  "result": "1-2",  // 更新为实际比分
  "status": "已结束"  // 更新为已结束
}
```

## 三、确认赛事报告链接

### 1. 一线队赛事报告链接

**文件路径**: `public/schedule.tsx`

**验证要点**:

- 一线队赛事报告使用 `match-report-v2.html` 展示
- 链接格式: `match-report-v2.html?date=YYYY-MM-DD&type=赛事类型&round=第X轮`
- 动态读取的JSON文件路径: `public/data/YYYY-MM-DD-赛事类型-第X轮.json`

### 2. B队赛事报告链接

**文件路径**: `public/schedule.tsx`

**验证要点**:

- B队赛事报告使用 `match-report.html` 展示

### 2. 验证比赛报告页面

**验证步骤**:

1. 启动开发服务器: `npm run dev`
2. 在浏览器中访问赛程页面
3. 点击更新后的比赛比分链接
4. 验证赛事报告页面是否正确加载
5. 检查以下内容是否显示正常:
   - 比赛基本信息
   - 球员阵容
   - 比赛统计
   - 比赛亮点
   - 关键因素

## 五、完整操作流程总结

1. **更新赛程文件**:
   - 一线队: 更新 `schedule.json`
   - B队: 更新 `schedule_b.json`
2. **验证赛事报告链接**:
   - 确认 `schedule.tsx` 中的链接格式正确
   - 确认动态加载的JSON文件路径正确
3. **验证赛事报告**:
   - 确认JSON文件结构完整
   - 在浏览器中验证赛事报告页面显示正常

## 六、注意事项

1. 球队名称必须与JSON文件中的名称完全一致
2. 比分格式必须为 "X-Y" 格式
3. 比赛状态必须更新为 "已结束"
4. 确保对应的比赛数据文件存在且格式正确
5. 在发布前进行本地验证，确保赛事报告能正常显示

## 七、故障排除

如果赛事报告无法正常显示，请检查:

1. JSON文件路径是否正确
2. JSON文件格式是否符合要求
3. 球队名称是否一致
4. URL参数格式是否正确
5. 开发服务器是否正常运行
