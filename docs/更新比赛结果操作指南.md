# 更新比赛结果操作指南

本文档详细说明如何更新上海海港足球俱乐部比赛结果的操作流程，包括一线队和B队的比赛结果更新。

## 一、一线队比赛结果更新流程

### 1. 更新赛程文件

**文件路径**: `public/data/schedule.json`

**操作步骤**:

1. 打开 `schedule.json` 文件
2. 找到对应的轮次（round字段）
3. 核实给出的两个球队名称是否与文件中的名称一致
4. 更新 `result` 字段为实际比分（格式："X-Y"）
5. 将 `status` 字段从 "未开始" 或其他状态改为 "已结束"

**示例**:

```json
{
  "id": 1,
  "round": "第1轮",
  "date": "2026-03-07",
  "homeTeam": "上海海港",
  "awayTeam": "河南队",
  "result": "1-2",  // 更新为实际比分
  "status": "已结束"  // 更新为已结束
}
```

### 2. 球员信息核对与标准化处理

**文件路径**:

- 球员名单参考文件: `public/data/players.json`
- 比赛数据文件: `public/data/YYYY-MM-DD-赛事类型-第X轮.json`

**核对步骤**:

1. 打开 `players.json` 文件，查看上海海港本赛季官方球员名单
2. 打开对应的比赛数据文件，检查以下内容:
   - 首发球员名单中的球员名字是否与 `players.json` 一致
   - 替补球员名单中的球员名字是否与 `players.json` 一致
   - 球员号码是否与 `players.json` 匹配
   - 球员位置是否与 `players.json` 匹配

**常见不一致情况**:

- 球员名字拼写不同（如：科乔·阿齐安贝 vs 让克劳德）
- 球员名字全称与简称的差异（如：马特乌斯·维塔尔 vs 维塔尔）
- 球员名字后缀差异（如：吾米提江·玉素普 vs 吾米提江）
- 球员位置信息不符

**修正方法**:

1. 以 `players.json` 中的球员信息为准
2. 更新比赛数据文件中的球员名字、号码和位置
3. 同时更新比赛时间线、技术统计和评分部分的球员名称
4. 确保所有相关字段都保持一致

### 3. 赛事报告汉化和标准化处理

**处理内容**:

1. **球员名称标准化**:
   - 使用 `players.json` 中的官方球员名称
   - 修正外籍球员的中文译名
   - 统一球员名称格式（避免全名与简称混用）
2. **场地名称汉化**:
   - 将英文场地名称翻译为中文
   - 例如："SAIC Motor Pudong Arena" → "上汽浦东足球场"
3. **赛事名称标准化**:
   - 使用规范的赛事名称
   - 例如："Chinese Super League" → "中国足球协会超级联赛"
4. **时间线描述标准化**:
   - 修正时间线中的球员名称
   - 统一事件描述格式
   - 确保描述与实际事件一致

**处理步骤**:

1. 打开比赛数据文件，检查以下字段:
   - `match.venue`（场地名称）
   - `match.competition`（赛事名称）
   - `lineups.home.players`（主队球员）
   - `lineups.home.substitutes`（主队替补）
   - `lineups.away.players`（客队球员）
   - `lineups.away.substitutes`（客队替补）
   - `matchTimeline`（比赛时间线）
   - `detailedPlayerStats`（详细球员统计）
   - `playerRatings`（球员评分）
2. 对照 `players.json` 进行修正:
   - 替换球员名称为官方名称
   - 修正场地名称为中文
   - 统一赛事名称格式
   - 更新时间线中的球员名称和描述
3. 验证所有相关字段的一致性:
   - 确保同一球员在不同部分的名称一致
   - 确保事件描述与实际情况一致
   - 确保所有汉化内容准确无误

### 4. 验证比赛报告页面

**验证步骤**:

1. 启动开发服务器: `npm run dev`
2. 在浏览器中访问赛程页面
3. 点击更新后的比赛比分链接
4. 验证赛事报告页面是否正确加载
5. 检查以下内容是否显示正常:
   - 比赛基本信息
   - 球员阵容（确保球员名字与官方名单一致）
   - 比赛统计
   - 比赛亮点
   - 关键因素

## 五、当季数据统计更新

### 1. 增量更新（推荐）

**文件路径**: `scripts/update_stats.py`

**操作步骤**:

1. 打开终端，进入项目根目录
2. 运行增量更新命令:
   ```bash
   python scripts/update_stats.py --incremental
   ```

**特点**:

- 只处理修改时间晚于上次更新时间的文件
- 速度快，适合日常赛事更新
- 自动判断是否需要更新

### 2. 完整更新

**操作步骤**:

1. 打开终端，进入项目根目录
2. 运行完整更新命令:
   ```bash
   python scripts/update_stats.py --full
   ```

**特点**:

- 重新统计所有比赛数据
- 适合数据修复或首次统计

## 六、补充赛程详细信息

### 1. 提取赛事报告数据

在更新比赛结果后，需要从对应的赛事报告JSON文件中提取详细信息并补充到赛程文件中。

**提取字段**:

| 字段 | 说明 | 数据来源 |
|------|------|----------|
| `referee` | 主裁判姓名 | 赛事报告的 `officials.referee` |
| `home_coach` | 主队主教练 | 赛事报告的 `lineups.home.manager` |
| `away_coach` | 客队主教练 | 赛事报告的 `lineups.away.manager` |
| `attendance` | 观众人数 | 赛事报告的 `match.attendance` |
| `scorers` | 进球者信息 | 赛事报告的 `highlights` 或 `matchTimeline`，需根据 `goal_type` 字段添加特殊标记 |

### 2. 自动更新脚本

**脚本路径**: `scripts/update_schedule_details_v2.py`

**操作步骤**:

1. 确保已更新赛程文件中的比赛结果
2. 运行更新脚本:
   ```bash
   python scripts/update_schedule_details_v2.py
   ```

**功能说明**:
- 自动匹配赛程与对应的赛事报告
- 提取裁判、主教练、观众人数信息
- 从 `highlights` 或 `matchTimeline` 中提取进球者
- 同时更新一线队和B队赛程文件

- 观众人数必须为数字格式

### 4. 进球者特殊标记处理

在提取进球者信息时，需要根据 `goal_type` 字段识别特殊进球类型，并添加相应标记。

**特殊进球类型标记**:

| goal_type | 标记 | 说明 |
|-----------|------|------|
| `penalty_goal` / `penalty` | `(PK)` | 点球进球 |
| `own_goal` | `(OG)` | 乌龙球 |

**处理步骤**:

1. 从赛事报告的 `highlights` 或 `matchTimeline` 中提取进球信息
2. 检查每个进球的 `goal_type` 字段
3. 根据 `goal_type` 添加相应标记：
   - 点球进球：`球员姓名(PK)`
   - 乌龙球：`球员姓名(OG)`
4. 如果是同一球员进了多个球（包括点球和乌龙球），需重复列出

**示例**:

```json
{
  "scorers": {
    "home": ["武磊", "武磊(PK)"],
    "away": ["克雷桑", "邓嘉俊(OG)"]
  }
}
```

**数据来源字段说明**:

| 字段路径 | 说明 |
|----------|------|
| `highlights[].goal_type` | 进球类型（用于判断点球或乌龙） |
| `highlights[].type == 'goal'` | 进球事件 |
| `highlights[].player` | 进球球员姓名 |
| `matchTimeline[].goal_type` | 同上 |
| `matchTimeline[].type == 'goal'` | 同上 |

**手动检查命令**:

```bash
# 检查所有已结束比赛的进球者与比分是否匹配
python -c "
import json
with open('public/data/schedule.json', 'r', encoding='utf-8') as f:
    schedule = json.load(f)
for match in schedule:
    if match.get('status') == '已结束':
        result = match.get('result', '')
        scorers = match.get('scorers', {})
        home_goals = len(scorers.get('home', []))
        away_goals = len(scorers.get('away', []))
        try:
            h, a = result.split('-')
            if int(h) != home_goals or int(a) != away_goals:
                print(f'进球数不匹配: {match.get(\"date\")} {match.get(\"homeTeam\")} {result}')
        except: pass
"
```

## 七、完整操作流程总结

1. **更新赛程文件**:
   - 一线队: 更新 `schedule.json` 中的 `result` 和 `status`
   - B队: 更新 `schedule_b.json` 中的 `result` 和 `status`
2. **验证赛事报告链接**:
   - 确认 `schedule.tsx` 中的链接格式正确
   - 确认动态加载的JSON文件路径正确
3. **球员信息核对**:
   - 参考 `players.json` 中的官方球员名单
   - 检查并修正比赛数据文件中的球员信息
   - 确保球员名字、号码和位置与官方名单一致
4. **补充赛程详细信息**:
   - 运行 `python scripts/update_schedule_details_v2.py`
   - 验证 `referee`、`home_coach`、`away_coach`、`attendance`、`scorers` 字段
5. **更新当季数据统计**:
   - **日常更新**: 使用增量更新 `python scripts/update_stats.py --incremental`
   - **数据修复**: 使用完整更新 `python scripts/update_stats.py --full`
6. **验证赛事报告**:
   - 确认JSON文件结构完整
   - 在浏览器中验证赛事报告页面显示正常

## 八、注意事项

1. 球队名称必须与JSON文件中的名称完全一致
2. 比分格式必须为 "X-Y" 格式
3. 比赛状态必须更新为 "已结束"
4. 确保对应的比赛数据文件存在且格式正确
5. 在发布前进行本地验证，确保赛事报告能正常显示

## 九、故障排除

如果赛事报告无法正常显示，请检查:

1. JSON文件路径是否正确
2. JSON文件格式是否符合要求
3. 球队名称是否一致
4. URL参数格式是否正确
5. 开发服务器是否正常运行
