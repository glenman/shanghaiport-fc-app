# Match Result Updater Skill

## 概述

本技能用于自动更新上海海港足球俱乐部的比赛结果，包括一线队和B队。

## 触发条件

当用户输入以下类似指令时触发：
- "更新XX队比赛结果"
- "update match result for XX team"
- "更新上海海港比赛结果"

## 功能特性

### 1. 更新赛程文件
- 更新一线队赛程：`public/data/schedule.json`
- 更新B队赛程：`public/data/schedule_b.json`
- 设置比分和状态字段

### 2. 验证赛事报告
- 检查对应的JSON数据文件是否存在
- 验证球员姓名与 `players.json` 一致
- 检查汉化完整性

### 3. 更新数据统计
- 默认运行增量更新
- 支持全量更新

## 使用方法

### 输入格式

更新<球队类型>比赛结果，<主队>vs<客队> <比分>

### 示例

更新一线队比赛结果，上海海港vs武汉三镇 4:0

更新B队比赛结果，山西崇德荣海vs上海海港富盛经开 0:3

## 操作流程

### 步骤1：识别球队类型
- 根据用户输入判断是一线队还是B队

### 步骤2：定位比赛
- 在赛程文件中搜索匹配的球队名称
- 使用轮次或日期确定正确的比赛

### 步骤3：更新赛程
- 设置 `result` 字段为提供的比分
- 将 `status` 从 "未开始" 更改为 "已结束"

### 步骤4：验证赛事报告
- 检查对应的JSON文件是否存在
- 验证球员姓名是否与 `players.json` 一致
- 检查汉化完整性

### 步骤5：更新统计数据
- 默认运行增量更新
- 如果需要，提示用户进行全量更新

## 修改的文件

| 文件 | 说明 |
|------|------|
| `public/data/schedule.json` | 一线队赛程 |
| `public/data/schedule_b.json` | B队赛程 |
| `public/data/current_stats.json` | 当季统计数据 |

## 验证检查

1. **球队名称**：必须与赛程文件中的名称一致
2. **比分格式**：必须为 "X-Y" 格式
3. **比赛存在**：必须在赛程中找到匹配的比赛
4. **JSON文件**：对应的赛事报告JSON必须存在

## 注意事项

- 如果发现球员姓名不一致，会报告差异
- 默认使用增量更新以提高性能
- 在进行更改前会提示确认

## 手动执行步骤

如果需要手动执行，可以按照以下步骤：

1. **更新赛程**：
   - 编辑 schedule.json 或 schedule_b.json
   - 设置 result 和 status 字段

2. **验证赛事报告**：
   - 检查对应的 JSON 文件是否存在

3. **更新统计**：
   ```bash
   # 增量更新（推荐）
   python scripts/update_stats.py --incremental
   
   # 全量更新
   python scripts/update_stats.py --full
   ```

4. **验证**：
   - 启动开发服务器验证赛事报告显示正常

## 故障排除

### 问题：比赛结果未显示
- 检查赛程文件中的比分格式是否正确
- 确认状态已设置为 "已结束"
- 验证JSON文件路径是否正确

### 问题：球员姓名显示错误
- 检查 `players.json` 中的官方名称
- 确保赛事报告中的球员姓名与官方名单一致

### 问题：统计数据未更新
- 确认运行了更新脚本
- 检查文件修改时间是否正确
- 尝试使用全量更新

## 总结

本技能提供了便捷的比赛结果更新功能，自动完成赛程更新、数据验证和统计更新等一系列操作，确保数据的准确性和一致性。
