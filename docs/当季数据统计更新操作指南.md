# 当季数据统计更新操作指南

## 概述

本文档详细说明如何使用 `update_stats.py` 脚本更新上海海港足球俱乐部的当季数据统计，包括一线队和B队的各项数据。

## 脚本功能

- **全部重新抓取**：重新统计所有已结束的比赛数据
- **增量抓取**：只统计上次更新后新添加或修改的比赛数据
- **自动检测**：根据文件修改时间自动判断哪些是新的赛事报告

## 脚本位置

```
scripts/update_stats.py
```

## 运行环境

- Python 3.6+
- 无需额外依赖

## 使用方法

### 1. 全部重新抓取

当需要重新统计所有比赛数据时使用此选项：

```bash
python scripts/update_stats.py --full
```

### 2. 增量抓取

当只需要统计新的赛事报告时使用此选项：

```bash
python scripts/update_stats.py --incremental
```

### 3. 默认模式（全部重新抓取）

不指定参数时，默认执行全部重新抓取：

```bash
python scripts/update_stats.py
```

## 数据来源

脚本会从以下位置读取比赛数据：

```
public/data/
```

脚本会处理以下类型的文件：
- 中超比赛报告（如：2026-03-07-中超-第1轮.json）
- 中乙比赛报告（如：2026-03-22-中乙-第1轮.json）

## 数据输出

统计结果会保存到：

```
public/data/current_stats.json
```

## 统计内容

### 一线队（中超）
- 比赛场次
- 战绩（胜/平/负）
- 积分
- 进球数
- 失球数
- 进球榜
- 助攻榜
- 黄牌榜
- 红牌榜

### B队（中乙）
- 比赛场次
- 战绩（胜/平/负）
- 积分
- 进球数
- 失球数
- 进球榜
- 助攻榜
- 黄牌榜
- 红牌榜

## 数据更新逻辑

1. **增量抓取**：
   - 读取 `current_stats.json` 中的 `lastUpdated` 字段
   - 只处理修改时间晚于上次更新时间的文件
   - 自动判断是否需要全部重新抓取

2. **全部重新抓取**：
   - 处理所有符合条件的比赛文件
   - 重新计算所有统计数据

## 注意事项

1. **数据格式要求**：
   - 比赛文件必须包含 `match` 字段
   - 比赛状态必须为 `已结束`
   - 必须包含有效的比分结果

2. **红黄牌数据**：
   - 脚本从 `matchTimeline` 中提取红黄牌事件
   - 确保赛事报告中包含完整的 `matchTimeline` 数据

3. **球员姓名映射**：
   - 脚本会自动处理球员姓名的简写和变体
   - 例如："克劳德" → "让克劳德"，"让·克劳德" → "让克劳德"

## 错误处理

- 如果无法解析上次更新时间，脚本会自动执行全部重新抓取
- 如果没有找到符合条件的比赛数据，脚本会提示相应信息
- 如果保存文件失败，脚本会显示错误信息

## 常见问题

### Q: 为什么增量抓取没有统计到新的比赛数据？

A: 请检查：
1. 新的比赛文件是否已保存到 `public/data/` 目录
2. 比赛文件的修改时间是否晚于上次更新时间
3. 比赛状态是否设置为 `已结束`
4. 比赛文件是否包含有效的比分结果

### Q: 为什么红黄牌数据统计不正确？

A: 请检查：
1. 赛事报告中是否包含完整的 `matchTimeline` 数据
2. `matchTimeline` 中是否包含正确的红黄牌事件
3. 事件的 `team` 字段是否正确设置（home/away）

### Q: 如何手动触发全部重新抓取？

A: 使用 `--full` 参数：
```bash
python scripts/update_stats.py --full
```

## 扩展建议

1. **添加新赛事**：
   - 在 `COMPETITIONS` 字典中添加新的赛事类型
   - 确保新赛事的文件命名格式与现有格式一致

2. **自定义统计规则**：
   - 修改 `extract_statistics` 函数可以添加新的统计指标
   - 修改 `calculate_record` 函数可以调整战绩计算逻辑

3. **定时更新**：
   - 可以设置定时任务，定期执行增量抓取
   - 例如：每天凌晨自动运行增量抓取脚本

## 总结

`update_stats.py` 脚本提供了便捷的当季数据统计更新功能，支持全部重新抓取和增量抓取两种模式，能够根据文件修改时间智能判断新的赛事报告，确保数据统计的及时性和准确性。
