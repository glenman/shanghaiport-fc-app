# 历史赛季赛事报告优化指南

## 概述

本文档说明如何对历史赛事报告 JSON 文件进行汉化和标准化处理。通过总结具体赛季的处理经验，形成一套通用的优化工作流程，适用于2019、2020、2021、2022、2023等各个赛季。

## 通用优化流程

对任何一个赛季进行优化处理时，都需要遵循以下通用流程：


## 第一阶段：数据准备

### 1.1 准备数据文件

处理任何赛季前，需要准备以下数据文件：

**球员大名单文件**
- 路径：`datafile/上海海港XXXX一线队大名单.csv`
- 内容：球员的中英文名称对照
- 格式：
```csv

奥斯卡,Oscar,巴西,中场,28
佩雷拉,Vítor Pereira,葡萄牙,主教练
```

**赛程文件**
- 路径：`datafile/上海海港XXXX一线队中超赛程.csv`
- 内容：比赛日期、主客队对应关系
- 格式：
```csv
轮次,比赛日期,星期,时间,主队,客队,比分 (主:客),赛果 (海港视角)
Regular season,2020-07-27,星期一,19:35,天津泰达,上海上港,1:3,胜
```

**比赛结果文件**（用于核对）
- 路径：`datafile/上海海港XXXX中超联赛比赛结果.csv`
- 内容：比赛结果、进球队员
- 格式：
```csv
比赛类别,轮次,比赛时间,主队,比分,,客队,赛果,进球队员
中超联赛,1,2019.03.01,上海绿地,0,4,上海上港,胜,吕文君，奥斯卡
```

## 第三阶段：优化执行

### 3.1 队名修复（通用模板）

队名修复是最常见的优化操作。以下是通用处理逻辑：

```python
def fix_team_names_generic(json_path, schedule):
    """通用队名修复函数"""
    with open(json_path, 'r', encoding='utf-8') as f:
        data = json.load(f)

    date = data['match_info']['date']

    # 从赛程获取该日期的正确主客队
    if date in schedule:
        sched = schedule[date]
        correct_home = normalize_team_name(sched['home'])
        correct_away = normalize_team_name(sched['away'])

        # 更新队名
        data['teams']['home']['name'] = correct_home
        data['teams']['home']['full_name'] = correct_home
        data['teams']['away']['name'] = correct_away
        data['teams']['away']['full_name'] = correct_away

    # 清理Formation中的队名
    for team_key in ['home', 'away']:
        formation = data['teams'][team_key].get('formation', '')
        if '(' in formation:
            # 只保留阵型部分
            match = re.search(r'\(([\d-]+)\)', formation)
            if match:
                data['teams'][team_key]['formation'] = f"({match.group(1)})"

    with open(json_path, 'w', encoding='utf-8') as f:
        json.dump(data, f, ensure_ascii=False, indent=2)
```

### 3.3 球员姓名汉化

基于大名单建立球员映射：

```python
def build_player_map_from_roster(csv_path):
    """从CSV大名单建立球员映射"""
    player_map = {}

    with open(csv_path, 'r', encoding='utf-8-sig') as f:
        reader = csv.DictReader(f)
        for row in reader:
            name_cn = row.get('中文名称', '').strip()
            name_en = row.get('英文名称', '').strip()

            if name_cn and name_en:
                # 英文 -> 中文
                player_map[name_en] = name_cn

                # 处理特殊字符
                player_map[name_en.replace('á', 'a')] = name_cn
                player_map[name_en.replace('ć', 'c')] = name_cn

    return player_map
```

## 赛季处理经验总结

### 2019赛季

**特殊问题：**
- 文件名轮次与实际比赛日期不匹配（11个文件需要重命名）
- 主客队名称显示为"Opponent"

**处理流程：**
1. `fix_2019_filenames_and_rounds.py` - 修复文件名和round字段
2. `fix_2019_team_names.py` - 修复队名
3. `optimize_2019_reports.py` - 内容汉化

**文件数量：** 30轮

### 2020赛季

**特殊问题：**
- 赛会制，20轮比赛
- 主客队名称为英文（Shanghai SIPG等）
- Formation包含队名

**处理流程：**
1. `optimize_2020_reports.py` - 一站式完成所有优化

**文件数量：** 20轮

### 2021-2023赛季

**特点：**
- 文件名和轮次一般正确
- 主要问题是球员姓名和队名汉化

**处理流程：**
1. `optimize_XXXX_reports.py` - 球员和队名汉化
2. `fix_XXXX_substitutions.py` - 换人信息修复（如需要）

## 故障排除

| 问题 | 可能原因 | 解决方案 |
|------|---------|---------|
| 球员未汉化 | 大名单中无该球员 | 手动添加到PLAYER_NAME_MAP |
| 队名仍是Opponent | 赛程CSV日期不匹配 | 检查日期格式是否一致 |
| Formation未清理 | 正则匹配失败 | 检查Formation格式 |
| 编码错误 | CSV文件编码不是UTF-8 | 使用encoding='utf-8-sig' |
| 文件名错误 | 轮次计算错误 | 参考赛程重新核对 |
