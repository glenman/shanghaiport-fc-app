# 历史赛季赛事报告优化指南

## 概述

本文档说明如何使用各赛季的优化脚本来对历史赛事报告 JSON 文件进行汉化和标准化处理。

## 目录结构

```
scripts/
├── optimize_2021_reports.py  # 2021赛季优化脚本
├── optimize_2022_reports.py  # 2022赛季优化脚本
├── optimize_2023_reports.py  # 2023赛季优化脚本
├── fix_2021_substitutions.py  # 2021赛季换人信息修复脚本
├── fix_2022_substitutions.py  # 2022赛季换人信息修复脚本
└── fix_team_names.py         # 队名修复脚本（去除Opponent问题）

datafile/
├── 上海海港2021一线队大名单.csv
├── 上海海港2022一线队大名单.csv
└── 上海海港2023一线队大名单.csv
```

## 功能说明

### 1. 球员姓名对照

脚本基于对应赛季的 `上海海港XXXX一线队大名单.csv` 文件创建球员姓名对照字典，支持以下两种匹配方式：

#### 按号码匹配
- 优先根据球衣号码进行精确匹配
- 适用于号码唯一的情况

#### 按拼音匹配
- 当号码匹配失败时，根据球员姓名的拼音进行模糊匹配
- 支持拼音部分匹配和包含关系
- 自动处理常见的拼音分隔符（-、·、'等）

### 3. 内容优化

脚本会自动优化以下内容：

#### 比赛信息 (match_info)
- 赛事名称汉化：`Chinese Football Association Super League` → `中国足球协会超级联赛`
- 轮次汉化：`Matchweek 1` → `第1轮`
- 场地名称汉化：将英文场地名称翻译为中文

#### 球队信息 (teams)
- 球队名称汉化
- 球队全名生成
- 阵容球员姓名汉化（基于号码和拼音）
- 替补球员姓名汉化
- 换人记录中的球员姓名汉化
- 国籍标准化：`Unknown` → `中国`
- **队名修复**：修正错误的"Opponent"为正确的球队名称
- **Formation清理**：去除formation中的球队名称，只保留阵型

#### 比赛事件 (events)
- 进球、黄牌、红牌等事件中的球员姓名汉化
- **换人事件补全**：补充substitution事件的player2字段（下场球员）

#### 球员统计 (player_stats)
- 球员个人统计数据中的姓名汉化

#### 换人信息 (substitutions)
- **teams.substitutions**：确保player_out和player_in字段都正确填充
- **events中的substitution**：确保player（上场）和player2（下场）字段都正确填充

当发现主客队名称出现"Opponent"等错误时，可使用队名修复脚本：

```bash

```

该脚本会自动：
1. 从formation字段中提取正确的球队名称
2. 替换teams.home.name和teams.away.name中的错误名称
3. 清理formation字段中的球队名称，只保留阵型

脚本会自动：
1. 扫描对应赛季目录下的所有JSON文件（如 `public/data/history/2021/`）
2. 对每个文件进行优化处理
3. 显示处理进度和结果统计

## 数据文件说明

### 球员名单文件

**文件路径：** `datafile/上海海港XXXX一线队大名单.csv`

**文件格式：**
```csv
中文名称,英文名称,国籍,位置,年龄
颜骏凌,Yan Junling,中国,GK,31
李昂,Li Ang,中国,DF,28
...
```

**注意事项：**
- 号码必须唯一
- 姓名使用中文
- 位置使用标准缩写（GK、DF、MF、FW）
- 拼音字段用于模糊匹配

### JSON文件

**文件路径：** `public/data/history/XXXX/*.json`

**文件命名格式：** `YYYY-MM-DD-中超-第N轮.json`

## 换人信息修复详解

### 问题描述

原始数据中，换人信息存在不完整的问题：
1. `teams[home/away].substitutions` 中只有 `player_out`，缺少 `player_in`
2. `events` 中 type 为 "substitution" 的事件只有 `player`（上场球员），缺少 `player2`（下场球员）

### 修复逻辑

1. **从events提取player_in**：遍历events中的substitution事件，提取上场球员名称
2. **更新teams.substitutions.player_in**：根据时间和球队匹配，填充player_in字段
3. **从teams提取player_out**：遍历teams.substitutions，提取下场球员名称
4. **更新events.player2**：根据时间和球队匹配，填充player2字段

### 前端显示

前端 `history-match-report.html` 中使用以下字段显示换人信息：
- `event.player2` 或 `event.player_out` - 下场球员
- `event.player` - 上场球员

## 注意事项

1. **备份原文件**
   - 脚本会直接修改JSON文件
   - 建议在运行前备份对应赛季目录

2. **数据一致性**
   - 确保球员名单文件中的号码与JSON文件中的号码一致
   - 检查拼音拼写是否正确

3. **编码问题**
   - 脚本使用UTF-8编码
   - 确保所有文件使用正确的编码

4. **测试验证**
   - 运行脚本后，随机抽查几个JSON文件
   - 检查球员和球队名称是否正确汉化
   - 启动开发服务 `npm run dev` 验证页面显示

## 故障排除

### 常见问题

**问题1：球员名称未汉化**
- 检查球员名单CSV文件中是否有该球员
- 检查号码是否正确
- 检查拼音是否正确

**问题2：球队名称显示为"Opponent"**
- 运行队名修复脚本：`python scripts/fix_team_names.py`
- 检查formation字段是否包含正确的球队名称

**问题3：换人信息显示"undefined"**
- 运行换人信息修复脚本
- 检查teams.substitutions和events中的字段是否完整

**问题4：编码错误**
- 确保CSV文件使用UTF-8编码
- 确保JSON文件使用UTF-8编码
