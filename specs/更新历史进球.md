# 更新历史进球

## 脚本说明

本目录包含用于自动化处理足球数据的脚本，主要用于从上海海港一线队的比赛报告中提取进球信息并更新到 `goal_details.json` 文件中。

## 可用脚本

### `auto_update_goal_details.py`

**功能**：自动扫描上海海港一线队的比赛报告文件，提取进球信息，并更新到 `goal_details.json` 文件中。

**特点**：
- 自动扫描 `public/data` 目录下的中超比赛报告
- 只处理上海海港一线队的比赛，不处理B队
- 自动检测重复记录，避免重复添加
- 自动生成正确的比赛结果（胜、负、平）
- 自动分配ID，确保ID连续递增

**使用方法**：
```bash
python scripts/auto_update_goal_details.py
```

**运行结果示例**：

## 定期更新流程

### 步骤 1：更新比赛报告
1. 在 `public/data` 目录下添加或更新上海海港一线队的比赛报告文件
2. 确保比赛报告文件包含 `matchTimeline` 字段，其中记录了进球事件

### 步骤 2：运行自动化脚本
```bash
python scripts/auto_update_goal_details.py
```

### 步骤 3：验证结果
脚本会自动：
- 扫描所有中超比赛报告文件
- 提取上海海港一线队的进球信息
- 检查是否已存在相同记录（通过日期、球员和进球时间）
- 添加新的进球记录到 `goal_details.json`

## 文件格式要求

比赛报告文件需要包含以下字段：
- `match`: 包含基本比赛信息（date, homeTeam, awayTeam, result, round, season）
- `matchTimeline`: 包含比赛时间线，其中 `type: "goal"` 的事件会被识别为进球

## 注意事项

1. 脚本会自动处理重复记录，不会重复添加
2. 只支持中超联赛的比赛报告（不处理中乙联赛）
3. 只处理上海海港一线队的比赛（不处理B队）
4. 自动识别比赛结果（胜、负、平）
5. 自动分配ID，确保ID连续递增
6. 支持补时进球（如 45+4'）
